# 상품 관리 테스트 케이스

## TC-PROD-001: 상품 목록 조회 및 페이지네이션

- **우선순위**: High
- **유형**: 기능
- **사전조건**: ADMIN 계정으로 로그인됨. 상품 데이터가 존재함
- **테스트 절차**:
  1. `/products` 페이지 접속
  2. 페이지 로드 완료 대기
  3. 테이블 하단 페이지네이션 확인
- **기대결과**: 상품 목록이 테이블로 표시됨 (이미지, 상품명/설명, 브랜드, 카테고리, SKU, 가격, 상태, 생성일 컬럼). 상단에 총 상품수/활성/브랜드수/카테고리수 통계 카드 표시됨
- **관련 API**: `GET /admin/products` (page, size 파라미터)

---

## TC-PROD-002: 상품명 검색

- **우선순위**: High
- **유형**: 기능
- **사전조건**: 상품이 존재하며 이름을 알고 있음
- **테스트 절차**:
  1. ProductFilterBar 검색창에 상품명 일부 입력
  2. 결과 확인
- **기대결과**: 해당 키워드가 포함된 상품명의 상품만 표시됨. 검색어 변경 시 페이지가 0으로 리셋됨
- **관련 API**: `GET /admin/products/search?name={name}`

---

## TC-PROD-003: 브랜드 필터

- **우선순위**: Medium
- **유형**: 기능
- **사전조건**: 여러 브랜드의 상품이 존재함
- **테스트 절차**:
  1. ProductFilterBar에서 브랜드 필터 선택
  2. 특정 브랜드 선택
  3. 결과 확인
- **기대결과**: 선택한 브랜드의 상품만 목록에 표시됨
- **관련 API**: `GET /admin/products/brand/{brand}`

---

## TC-PROD-004: 상품 생성 성공

- **우선순위**: High
- **유형**: 기능
- **사전조건**: `/products` 페이지 접속됨
- **테스트 절차**:
  1. "상품 추가" 버튼 클릭
  2. 상품 생성 다이얼로그에서 상품명(필수) 입력
  3. SKU, 브랜드, 설명, 썸네일 URL, 기본 이미지 URL 입력 (선택)
  4. 저장 버튼 클릭
- **기대결과**: 다이얼로그가 닫히고 성공 Toast 표시. 상품 목록이 갱신되어 새 상품이 나타남
- **관련 API**: `POST /admin/products`

---

## TC-PROD-005: 상품명 누락 시 생성 실패

- **우선순위**: Medium
- **유형**: 에러처리
- **사전조건**: 상품 생성 다이얼로그가 열려 있음
- **테스트 절차**:
  1. 상품명 필드를 비워둔 채 저장 버튼 클릭
- **기대결과**: 폼 유효성 검사에 의해 제출이 차단됨. 상품명 필수 입력 에러 메시지 표시됨
- **관련 API**: 없음 (클라이언트사이드 유효성 검사)

---

## TC-PROD-006: 상품 수정

- **우선순위**: High
- **유형**: 기능
- **사전조건**: 수정할 상품이 존재함
- **테스트 절차**:
  1. 상품 목록에서 특정 상품의 액션 메뉴 클릭
  2. "수정" 선택
  3. 상품명 또는 브랜드 변경
  4. 저장 버튼 클릭
- **기대결과**: 성공 Toast 표시. 상품 목록이 갱신되어 변경된 정보가 반영됨
- **관련 API**: `PUT /admin/products`

---

## TC-PROD-007: 상품 삭제 - 확인 다이얼로그 및 성공

- **우선순위**: High
- **유형**: 기능
- **사전조건**: 삭제할 상품이 존재함
- **테스트 절차**:
  1. 상품 액션 메뉴에서 "삭제" 선택
  2. 확인 다이얼로그에서 상품명 확인
  3. "삭제" 버튼 클릭
- **기대결과**: 확인 다이얼로그에 상품명이 표시됨. 삭제 성공 Toast 표시. 목록에서 제거됨
- **관련 API**: `DELETE /admin/products/{id}`

---

## TC-PROD-008: 상품 상세 다이얼로그 조회

- **우선순위**: Medium
- **유형**: UI
- **사전조건**: 상품 데이터가 존재함
- **테스트 절차**:
  1. 상품 액션 메뉴에서 "상세보기" 선택
- **기대결과**: 상품 상세 다이얼로그가 열리고 이미지, SKU, 브랜드, 상품명, 설명, 썸네일, 기본 이미지, 생성일/수정일이 표시됨
- **관련 API**: `GET /admin/products/{id}`

---

## TC-PROD-009: 이미지가 없는 상품의 이미지 플레이스홀더 표시

- **우선순위**: Low
- **유형**: UI
- **사전조건**: 이미지 URL이 없는 상품이 존재함
- **테스트 절차**:
  1. 상품 목록에서 이미지가 없는 상품 행 확인
- **기대결과**: 이미지 셀에 이미지 아이콘(ImageIcon)이 있는 muted 배경 플레이스홀더가 표시됨
- **관련 API**: 없음

---

## TC-PROD-010: SKU로 상품 조회

- **우선순위**: Medium
- **유형**: 기능
- **사전조건**: SKU가 등록된 상품이 존재함
- **테스트 절차**:
  1. SKU 검색 기능을 통해 특정 SKU 입력 (구현된 경우)
- **기대결과**: 해당 SKU의 상품이 조회됨
- **관련 API**: `GET /admin/products/sku/{sku}`

> ⚠️ 미구현: 현재 UI에서 SKU 직접 검색 필드가 별도로 존재하지 않음. 일반 검색에서 처리되는지 확인 필요

---

## TC-PROD-011: 상품 필터 초기화

- **우선순위**: Low
- **유형**: UI
- **사전조건**: 검색어 및 브랜드 필터가 적용된 상태
- **테스트 절차**:
  1. 검색어와 브랜드 필터를 모두 적용
  2. 초기화 버튼 클릭
- **기대결과**: 모든 필터가 초기화되고 전체 상품 목록이 표시됨
- **관련 API**: `GET /admin/products`
