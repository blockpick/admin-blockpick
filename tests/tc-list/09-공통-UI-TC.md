# 공통 UI 테스트 케이스

## TC-UI-001: 사이드바 접기/펼치기

- **우선순위**: Medium
- **유형**: UI
- **사전조건**: 로그인된 상태에서 AdminLayout이 표시된 페이지
- **테스트 절차**:
  1. 사이드바의 접기/펼치기 버튼 클릭
  2. 다시 클릭하여 원래 상태로 복원
- **기대결과**: 사이드바가 축소되어 아이콘만 표시되거나, 완전히 숨겨짐. 메인 콘텐츠 영역이 사이드바 너비만큼 확장됨. 다시 클릭 시 원래 상태로 복원됨
- **관련 API**: 없음

## TC-UI-002: 모바일 반응형 - 햄버거 메뉴

- **우선순위**: Medium
- **유형**: UI
- **사전조건**: 브라우저 창을 모바일 크기(375px 이하)로 줄인 상태
- **테스트 절차**:
  1. 화면 상단의 햄버거 메뉴(☰) 버튼 클릭
  2. 네비게이션 항목 선택
- **기대결과**: 햄버거 메뉴 클릭 시 Sheet(슬라이드) 형태의 사이드 네비게이션이 나타남. 네비게이션 항목 선택 후 Sheet가 닫힘
- **관련 API**: 없음

## TC-UI-003: 모바일 반응형 - Sheet 사이드바 닫기

- **우선순위**: Low
- **유형**: UI
- **사전조건**: 모바일 크기에서 Sheet 사이드바가 열린 상태
- **테스트 절차**:
  1. Sheet 사이드바 외부 영역 클릭
  2. 또는 닫기(X) 버튼 클릭
- **기대결과**: Sheet가 닫히고 메인 콘텐츠가 표시됨
- **관련 API**: 없음

## TC-UI-004: 다크모드 토글

- **우선순위**: Low
- **유형**: UI
- **사전조건**: 로그인된 상태
- **테스트 절차**:
  1. 헤더 또는 설정 메뉴에서 다크모드 토글 버튼 클릭
  2. 라이트모드로 다시 전환
- **기대결과**: 전체 UI가 다크 테마로 변경됨. 재클릭 시 라이트 테마로 복원됨. 설정이 페이지 새로고침 후에도 유지됨 (localStorage 또는 시스템 설정)
- **관련 API**: 없음

## TC-UI-005: 네비게이션 메뉴 이동

- **우선순위**: High
- **유형**: UI
- **사전조건**: 로그인된 상태
- **테스트 절차**:
  1. 사이드바에서 "사용자 관리" 클릭
  2. "게임 관리" 클릭
  3. "상품 관리" 클릭
  4. "블록체인" 클릭
  5. "모니터링" 클릭
- **기대결과**: 각 클릭 시 해당 페이지(`/users`, `/games`, `/products`, `/blockchain`, `/monitoring`)로 이동됨. 현재 활성 메뉴 항목이 시각적으로 강조됨
- **관련 API**: 없음

## TC-UI-006: 빈 상태(EmptyState) 표시

- **우선순위**: Medium
- **유형**: UI
- **사전조건**: 데이터가 없거나 필터 결과가 없는 상태
- **테스트 절차**:
  1. 사용자 목록 페이지에서 결과가 없는 검색 조건 적용
  2. EmptyState 컴포넌트 확인
- **기대결과**: 해당 기능에 맞는 아이콘, 제목("사용자가 없습니다" 등), 설명 텍스트가 표시됨. 관련 액션 버튼(예: "사용자 추가")이 있는 경우 표시됨
- **관련 API**: 없음

## TC-UI-007: 로딩 스피너 표시

- **우선순위**: Medium
- **유형**: UI
- **사전조건**: API 응답 지연이 발생하는 상황
- **테스트 절차**:
  1. 네트워크 속도를 느리게 설정 (브라우저 DevTools > Network > Slow 3G)
  2. `/users` 또는 `/games` 페이지 접속
- **기대결과**: 데이터 로드 중 LoadingSpinner 컴포넌트가 표시됨. 데이터 로드 완료 후 스피너가 사라지고 데이터 테이블이 표시됨
- **관련 API**: 없음 (로딩 상태 처리)

## TC-UI-008: 토스트 알림 표시 및 자동 소멸

- **우선순위**: Medium
- **유형**: UI
- **사전조건**: 액션(생성/수정/삭제)이 성공한 상태
- **테스트 절차**:
  1. 사용자 생성 성공 후 Toast 알림 확인
  2. 일정 시간 대기
- **기대결과**: 화면 우하단 또는 우상단에 Toast 알림이 표시됨 (제목 + 설명). 약 3~5초 후 자동으로 사라짐. 성공은 기본색, 실패는 destructive(빨간색) 스타일
- **관련 API**: 없음

## TC-UI-009: 에러 상태 표시 (API 오류)

- **우선순위**: Medium
- **유형**: 에러처리
- **사전조건**: 네트워크 오류 또는 서버 오류 발생 상황
- **테스트 절차**:
  1. API 요청이 실패하도록 강제 (네트워크 오프라인 또는 잘못된 토큰)
  2. `/users` 페이지 접속
- **기대결과**: LoadingSpinner 대신 에러 EmptyState가 표시됨: "사용자를 불러올 수 없습니다" 메시지와 권한/로그인 확인 안내
- **관련 API**: 없음

## TC-UI-010: 확인 다이얼로그 공통 동작

- **우선순위**: Medium
- **유형**: UI
- **사전조건**: 삭제 또는 위험 작업을 트리거한 상태
- **테스트 절차**:
  1. 삭제 버튼 클릭으로 ConfirmDialog 열기
  2. 다이얼로그 내용 확인
  3. X 버튼 또는 배경 클릭으로 닫기
- **기대결과**: ConfirmDialog에 제목, 설명(대상 항목명 포함), 취소/확인 버튼이 표시됨. 취소 클릭 또는 외부 클릭 시 다이얼로그가 닫히고 작업이 취소됨. destructive 변형 시 확인 버튼이 빨간색으로 표시됨
- **관련 API**: 없음

## TC-UI-011: 페이지 헤더 및 액션 버튼

- **우선순위**: Low
- **유형**: UI
- **사전조건**: AdminLayout이 있는 페이지 접속됨
- **테스트 절차**:
  1. 각 페이지(사용자, 게임, 상품)의 헤더 영역 확인
- **기대결과**: PageHeader 컴포넌트가 제목, 설명, 주요 액션 버튼(예: "사용자 추가")을 포함하여 표시됨. 액션 버튼 클릭 시 해당 다이얼로그가 열림
- **관련 API**: 없음
