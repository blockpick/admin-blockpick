# 파일 업로드 테스트 케이스

## TC-FILE-001: 단일 이미지 업로드 성공 (JPG)

- **우선순위**: High
- **유형**: 기능
- **사전조건**: ADMIN 계정으로 로그인됨. `/storage` 페이지 접속됨
- **테스트 절차**:
  1. "파일 업로드" 버튼 클릭
  2. 업로드 다이얼로그에서 폴더를 "products"로 선택
  3. 파일 선택기에서 JPG 이미지 파일 1개 선택
- **기대결과**: 업로드 성공 Toast("파일이 업로드되었습니다") 표시됨. 다이얼로그가 닫힘. 선택된 폴더의 파일 그리드에 새 이미지가 표시됨. S3 URL 및 썸네일 URL이 반환됨
- **관련 API**: `POST /admin/storage/upload/image` (folder: products)

## TC-FILE-005: 지원하지 않는 파일 형식 업로드 시도

- **우선순위**: Medium
- **유형**: 에러처리
- **사전조건**: `/storage` 페이지 접속됨
- **테스트 절차**:
  1. "파일 업로드" 버튼 클릭
  2. PDF 또는 TXT 등 이미지가 아닌 파일 선택 시도
- **기대결과**: 파일 선택기의 `accept="image/*"` 속성으로 인해 비이미지 파일은 선택 불가. 또는 서버에서 에러 응답 시 에러 Toast 표시
- **관련 API**: `POST /admin/storage/upload/image` → 400 에러 (지원하지 않는 형식)

## TC-FILE-006: 폴더 선택 후 파일 목록 변경

- **우선순위**: Medium
- **유형**: 기능
- **사전조건**: 다양한 폴더에 파일이 존재함
- **테스트 절차**:
  1. `/storage` 페이지 접속
  2. 폴더 선택 드롭다운에서 "games" 선택
  3. 다시 "products"로 변경
- **기대결과**: 선택한 폴더의 파일만 그리드에 표시됨. 폴더 변경 시 페이지가 0으로 리셋됨. 통계 카드의 "폴더" 설명이 변경됨
- **관련 API**: `GET` 파일 목록 API (folder 파라미터)

## TC-FILE-007: 파일 상세 정보 다이얼로그

- **우선순위**: Medium
- **유형**: UI
- **사전조건**: 파일이 존재함
- **테스트 절차**:
  1. 파일 그리드에서 파일 카드에 마우스 오버
  2. 나타나는 버튼에서 "상세보기" 클릭
- **기대결과**: 파일 정보 다이얼로그가 열림. 이미지 미리보기(이미지인 경우), 파일명, 폴더, 크기, 타입, 업로드일, URL이 표시됨. URL 복사 버튼이 작동함
- **관련 API**: 파일 단건 조회 API

---

## TC-FILE-008: 파일 삭제

- **우선순위**: Medium
- **유형**: 기능
- **사전조건**: 삭제할 파일이 존재함
- **테스트 절차**:
  1. 파일 카드에서 "삭제" 메뉴 선택
  2. 확인 다이얼로그에서 "삭제" 버튼 클릭
- **기대결과**: 성공 Toast("파일이 삭제되었습니다") 표시됨. 그리드에서 해당 파일 카드가 제거됨
- **관련 API**: 파일 삭제 API

---

## TC-FILE-009: 이미지/비이미지 구분 렌더링

- **우선순위**: Low
- **유형**: UI
- **사전조건**: 이미지 파일과 비이미지 파일이 모두 존재함
- **테스트 절차**:
  1. `/storage` 페이지에서 파일 그리드 확인
- **기대결과**: 이미지 파일(`image/*` MIME 타입)은 썸네일 이미지로 표시됨. 비이미지 파일은 FileText 아이콘이 있는 muted 배경으로 표시됨
- **관련 API**: 없음

---

## TC-FILE-010: 빈 폴더 상태 표시

- **우선순위**: Low
- **유형**: UI
- **사전조건**: 비어 있는 폴더 선택
- **테스트 절차**:
  1. 파일이 없는 폴더 선택
- **기대결과**: EmptyState 컴포넌트가 표시됨: "{폴더명} 폴더에 파일이 없습니다" 메시지
- **관련 API**: 파일 목록 API → 빈 결과
