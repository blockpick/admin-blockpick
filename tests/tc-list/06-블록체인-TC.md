# 블록체인 관리 테스트 케이스

## TC-BC-001: 블록체인 컨트랙트 목록 조회

- **우선순위**: High
- **유형**: 기능
- **사전조건**: ADMIN 계정으로 로그인됨. 배포된 컨트랙트가 존재함
- **테스트 절차**:
  1. `/blockchain` 페이지 접속
  2. 페이지 로드 완료 대기
- **기대결과**: 컨트랙트 목록이 테이블로 표시됨 (게임명, 컨트랙트 주소, 네트워크, 상태, 배포일 컬럼). 상단에 총/배포성공/배포대기/배포실패 통계 카드 4개 표시됨
- **관련 API**: 게임 목록 및 컨트랙트 정보 조합

---

## TC-BC-002: 게임 생성 + 스마트 컨트랙트 배포

- **우선순위**: High
- **유형**: 기능
- **사전조건**: 블록체인 관리 또는 게임 생성 다이얼로그에서 배포 옵션이 활성화된 상태
- **테스트 절차**:
  1. 게임 생성 다이얼로그에서 "컨트랙트 배포" 옵션 활성화
  2. 게임 필수 정보(제목, 유형, 시작/종료시간 등) 입력
  3. 저장 버튼 클릭
- **기대결과**: 게임이 생성되고 블록체인 컨트랙트 배포가 요청됨. 성공 Toast 표시. 블록체인 페이지에서 새 컨트랙트 항목이 생성됨
- **관련 API**: `POST /admin/blockchain/games` (deployContract: true)

---

## TC-BC-003: 컨트랙트 배포 완료 처리

- **우선순위**: High
- **유형**: 기능
- **사전조건**: 배포 중인(DEPLOYING/PENDING) 게임이 존재함
- **테스트 절차**:
  1. 배포 완료 처리 API 호출 또는 UI 액션 실행
  2. 컨트랙트 주소와 배포 트랜잭션 해시 제공
- **기대결과**: 게임 상태가 IN_PROGRESS로 변경됨. 컨트랙트 상태가 DEPLOYED로 변경됨

> ⚠️ 미구현: 현재 UI에서 수동으로 배포 완료 처리하는 버튼이 없음. 웹훅/자동 처리로 동작할 것으로 보임
- **관련 API**: `POST /admin/blockchain/games/{gameId}/contract-deployed`

---

## TC-BC-004: 컨트랙트 배포 실패 처리

- **우선순위**: High
- **유형**: 에러처리
- **사전조건**: 컨트랙트 배포가 실패한 게임이 존재함
- **테스트 절차**:
  1. 컨트랙트 목록에서 FAILED 상태 컨트랙트 확인
  2. 관련 에러 정보 확인
- **기대결과**: 컨트랙트 상태가 FAILED로 표시됨. 게임 상태가 FAILED로 변경됨

> ⚠️ 미구현: 현재 UI에서 실패 처리 수동 버튼 없음
- **관련 API**: `POST /admin/blockchain/games/{gameId}/contract-deployment-failed`

## TC-BC-005: 참여 블록체인 상태 조회 (엔트리 ID 검색)

- **우선순위**: High
- **유형**: 기능
- **사전조건**: 블록체인 참여 기록이 존재하며 엔트리 ID를 알고 있음
- **테스트 절차**:
  1. `/blockchain` 페이지 접속
  2. "엔트리 ID 검색" 필드에 유효한 엔트리 ID 입력
  3. "검색" 버튼 클릭
- **기대결과**: 엔트리 상태 다이얼로그가 열림. 엔트리 ID, 상태(CONFIRMED/PENDING/FAILED), 트랜잭션 인텐트 목록이 표시됨
- **관련 API**: `GET /admin/blockchain/entries/{entryId}/status`

## TC-BC-006: 엔트리 ID 누락 시 검색 차단

- **우선순위**: Medium
- **유형**: 에러처리
- **사전조건**: `/blockchain` 페이지 접속됨
- **테스트 절차**:
  1. 엔트리 ID 검색 필드를 비워둔 채 "검색" 버튼 클릭 시도
- **기대결과**: 검색 버튼이 비활성화(disabled) 상태이거나, 클릭 시 에러 Toast("엔트리 ID를 입력해주세요") 표시됨
- **관련 API**: 없음

## TC-BC-007: 컨트랙트 수동 검증

- **우선순위**: Medium
- **유형**: 기능
- **사전조건**: 배포된(DEPLOYED) 컨트랙트가 존재함
- **테스트 절차**:
  1. 컨트랙트 목록에서 특정 항목의 액션 메뉴 클릭
  2. "컨트랙트 검증" 선택
- **기대결과**: 성공 Toast("컨트랙트가 성공적으로 검증되었습니다") 표시됨. Polygonscan 검증이 요청됨
- **관련 API**: `POST /admin/blockchain/games/{gameId}/verify-contract`

## TC-BC-008: 컨트랙트 주소 복사

- **우선순위**: Low
- **유형**: UI
- **사전조건**: 컨트랙트 주소가 있는 항목이 존재함
- **테스트 절차**:
  1. 컨트랙트 목록에서 주소 셀의 "복사" 버튼 클릭
  2. 또는 액션 메뉴에서 "주소 복사" 선택
- **기대결과**: 클립보드에 전체 컨트랙트 주소가 복사됨. Toast("컨트랙트 주소가 클립보드에 복사되었습니다") 표시됨
- **관련 API**: 없음

## TC-BC-009: 트랜잭션 해시 검색

- **우선순위**: Medium
- **유형**: 기능
- **사전조건**: 트랜잭션 해시를 알고 있음
- **테스트 절차**:
  1. "트랜잭션 해시 검색" 필드에 `0x...` 형식의 해시 입력
  2. "검색" 버튼 클릭 또는 Enter 키 입력
- **기대결과**: 트랜잭션 상세 다이얼로그가 열림. 해시, 상태, 블록 번호, 가스 사용량, 생성일이 표시됨
- **관련 API**: `GET /admin/blockchain/transactions/{txHash}`

## TC-BC-010: 컨트랙트 미배포 상태 표시

- **우선순위**: Low
- **유형**: UI
- **사전조건**: 컨트랙트가 배포되지 않은 게임이 존재함
- **테스트 절차**:
  1. 블록체인 페이지에서 NOT_DEPLOYED 상태 컨트랙트 확인
- **기대결과**: 상태 컬럼에 "미배포" Badge가 표시됨. 컨트랙트 주소와 배포일 컬럼에 "-"가 표시됨
- **관련 API**: 없음
