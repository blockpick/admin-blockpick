# 인증 테스트 케이스

## TC-AUTH-001: 관리자 로그인 성공

- **우선순위**: High
- **유형**: 기능
- **사전조건**: ADMIN 역할을 가진 계정이 존재함
- **테스트 절차**:
  1. `/login` 페이지 접속
  2. 이메일 필드에 `admin@blockpick.net` 입력
  3. 비밀번호 필드에 올바른 비밀번호 입력
  4. 로그인 버튼 클릭
- **기대결과**: `/dashboard` 페이지로 리다이렉트되고, localStorage에 `auth_token` 및 `refresh_token`이 저장됨
- **관련 API**: `POST /admin/auth/login`

---

## TC-AUTH-002: 잘못된 자격증명으로 로그인 실패

- **우선순위**: High
- **유형**: 에러처리
- **사전조건**: 없음
- **테스트 절차**:
  1. `/login` 페이지 접속
  2. 이메일 필드에 `wrong@example.com` 입력
  3. 비밀번호 필드에 잘못된 비밀번호 입력
  4. 로그인 버튼 클릭
- **기대결과**: 로그인 폼 하단에 에러 Alert가 표시되고, 페이지 이동 없음. 에러 메시지: "이메일 또는 비밀번호가 올바르지 않습니다"
- **관련 API**: `POST /admin/auth/login` → 401 INVALID_CREDENTIALS

## TC-AUTH-003: USER 역할 계정으로 로그인 시도 (403)

- **우선순위**: High
- **유형**: 보안
- **사전조건**: USER 역할을 가진 계정이 존재함
- **테스트 절차**:
  1. `/login` 페이지 접속
  2. USER 역할 계정의 이메일/비밀번호 입력
  3. 로그인 버튼 클릭
- **기대결과**: 에러 Alert 표시. 관리자 권한 없음 안내 메시지 출력. 대시보드로 이동 불가
- **관련 API**: `POST /admin/auth/login` → 403 FORBIDDEN

---

## TC-AUTH-004: Access Token 만료 후 자동 갱신

- **우선순위**: High
- **유형**: 기능
- **사전조건**: 로그인 상태이며 Access Token이 만료 직전 또는 만료됨. Refresh Token은 유효함
- **테스트 절차**:
  1. localStorage의 `auth_token`을 만료된 토큰으로 교체
  2. 보호된 페이지(예: `/users`)로 이동
  3. API 요청 발생 시 자동 토큰 갱신 과정 관찰
- **기대결과**: API 클라이언트가 자동으로 `/admin/auth/refresh`를 호출하여 새 Access Token을 발급받고, 원래 요청을 재시도하여 페이지가 정상 로드됨
- **관련 API**: `POST /admin/auth/refresh`

---

## TC-AUTH-005: Refresh Token 만료 시 로그인 리다이렉트

- **우선순위**: High
- **유형**: 기능
- **사전조건**: Access Token 및 Refresh Token 모두 만료됨
- **테스트 절차**:
  1. localStorage의 `auth_token`과 `refresh_token`을 모두 만료된 토큰으로 교체
  2. 보호된 페이지(예: `/games`)로 이동 또는 새로고침
- **기대결과**: 토큰 갱신 실패 후 자동으로 `/login` 페이지로 리다이렉트됨. localStorage에서 토큰이 삭제됨
- **관련 API**: `POST /admin/auth/refresh` → 401 INVALID_TOKEN

---

## TC-AUTH-006: 로그아웃 후 보호된 페이지 접근 불가

- **우선순위**: High
- **유형**: 보안
- **사전조건**: 로그인 상태
- **테스트 절차**:
  1. 사이드바 또는 프로필 메뉴에서 로그아웃 버튼 클릭
  2. 로그아웃 완료 확인
  3. `/dashboard` 또는 `/users` URL로 직접 접근 시도
- **기대결과**: localStorage의 `auth_token`, `refresh_token`이 삭제됨. 보호된 페이지 접근 시 `/login`으로 리다이렉트됨
- **관련 API**: 클라이언트사이드 로그아웃 (localStorage 클리어)

---

## TC-AUTH-007: 토큰 검증 (페이지 로드 시)

- **우선순위**: High
- **유형**: 기능
- **사전조건**: 유효한 Access Token이 localStorage에 저장되어 있음
- **테스트 절차**:
  1. 브라우저에서 `/dashboard` URL로 직접 접근
  2. 페이지 로드 중 네트워크 탭 관찰
- **기대결과**: `POST /admin/auth/verify` 요청이 발생하고 200 응답을 받아 관리자 정보가 로드됨. 페이지가 정상적으로 렌더링됨
- **관련 API**: `POST /admin/auth/verify`

---

## TC-AUTH-008: 이미 로그인된 상태에서 로그인 페이지 접근

- **우선순위**: Medium
- **유형**: UI
- **사전조건**: 유효한 토큰으로 로그인된 상태
- **테스트 절차**:
  1. 로그인 상태에서 `/login` URL로 직접 접근
- **기대결과**: `/dashboard` 페이지로 자동 리다이렉트됨. 로그인 폼이 표시되지 않음
- **관련 API**: 없음 (클라이언트사이드 리다이렉트)
