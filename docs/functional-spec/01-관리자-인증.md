# 관리자 인증

> 버전: 1.0 | 작성일: 2026-02-24

---

## 1. 개요

- **목적**: BlockPick 관리자 시스템 접근을 위한 인증 및 세션 관리
- **대상 사용자**: ADMIN, PARTNER 역할을 가진 운영자
- **관련 화면**: `/login`

---

## 2. 화면 구성

| 화면 | 경로 | 주요 기능 |
|------|------|----------|
| 로그인 페이지 | `/login` | 이메일/비밀번호 로그인, 에러 메시지 표시 |

---

## 3. API 목록

| # | API명 | 메서드 | 경로 | 설명 | 구현 |
|---|-------|--------|------|------|------|
| 1 | 관리자 로그인 | POST | `/admin/auth/login` | 이메일/비밀번호로 로그인, 토큰 발급 | ✅ |
| 2 | 토큰 갱신 | POST | `/admin/auth/refresh` | Refresh Token으로 새 Access Token 발급 | ✅ |
| 3 | 토큰 검증 | POST | `/admin/auth/verify` | Access Token 유효성 검증 및 관리자 정보 반환 | ✅ |
| 4 | 로그아웃 | POST | `/admin/auth/logout` | 서버 측 세션 무효화 | ✅ |
| 5 | 현재 사용자 조회 | GET | `/admin/auth/me` | 현재 로그인한 관리자 정보 반환 | ✅ |

## 4. 상세 기능

### 4.1 로그인 (`authService.login()`)

**설명**: ADMIN 또는 PARTNER 역할을 가진 사용자만 로그인 가능.

**입력 (Request Body):**
```json
{
  "email": "admin@blockpick.net",
  "password": "admin123!@#"
}
```

**출력 (Response 200):**
```json
{
  "success": true,
  "code": "SUCCESS",
  "message": "로그인 성공",
  "accessToken": "eyJhbGciOi...",
  "refreshToken": "eyJhbGciOi...",
  "admin": {
    "id": "uuid",
    "email": "admin@blockpick.net",
    "nickname": "관리자",
    "profileImageUrl": "https://...",
    "role": "ADMIN"
  }
}
```

**비즈니스 규칙:**
- `USER` 역할은 로그인 불가 (FORBIDDEN 403 반환)
- 로그인 성공 시 Access Token과 Refresh Token을 클라이언트에 저장 (쿠키 또는 localStorage)
- Access Token은 이후 모든 관리자 API 호출 시 `Authorization: Bearer <token>` 헤더로 전송

**수용 기준:**
- 올바른 자격증명으로 로그인 시 대시보드(`/`)로 리다이렉트
- 잘못된 자격증명 시 인라인 에러 메시지 표시
- USER 역할 계정 로그인 시 "접근 권한 없음" 메시지 표시

---

### 4.2 토큰 갱신 (`authService.refreshToken()`)

**설명**: Access Token 만료 시 Refresh Token으로 새 토큰 발급.

**입력 (Request Body):**
```json
{
  "refreshToken": "eyJhbGciOi..."
}
```

**출력 (Response 200):**
```json
{
  "success": true,
  "code": "SUCCESS",
  "message": "토큰 갱신 성공",
  "accessToken": "eyJhbGciOi..."
}
```

**비즈니스 규칙:**
- Access Token 만료 시 자동으로 Refresh Token으로 갱신 시도
- Refresh Token도 만료된 경우 로그인 페이지로 리다이렉트
- 갱신된 Access Token은 즉시 저장 후 원래 요청 재시도

---

### 4.3 토큰 검증 (`authService.verifyToken()`)

**설명**: 페이지 로드 시 현재 Access Token의 유효성을 검증하고 관리자 정보를 반환.

**입력 (Request Header):**
```
Authorization: Bearer eyJhbGciOi...
```

**출력 (Response 200):**
```json
{
  "success": true,
  "code": "SUCCESS",
  "message": "토큰 유효",
  "valid": true,
  "admin": {
    "id": "uuid",
    "email": "admin@blockpick.net",
    "nickname": "관리자",
    "profileImageUrl": null,
    "role": "ADMIN"
  }
}
```

**비즈니스 규칙:**
- 앱 초기 로드 시 또는 보호된 라우트 접근 시 호출
- `valid: false` 또는 에러 응답 시 로그인 페이지로 리다이렉트

---

### 4.4 로그아웃 (`authService.logoutServer()`)

**설명**: 서버 측에서 세션/토큰을 무효화하고 클라이언트 토큰을 삭제.

**비즈니스 규칙:**
- 서버 로그아웃 API 호출 후 클라이언트 측 토큰 삭제
- 로그아웃 완료 후 `/login` 페이지로 리다이렉트

## 6. 에러 처리

| 에러 코드 | HTTP 상태 | 원인 | UI 처리 |
|----------|----------|------|--------|
| `INVALID_CREDENTIALS` | 401 | 이메일 또는 비밀번호 불일치 | "이메일 또는 비밀번호가 올바르지 않습니다." |
| `FORBIDDEN` | 403 | USER 역할 계정 로그인 시도 | "관리자 권한이 없는 계정입니다." |
| `INVALID_TOKEN` | 401 | 만료되거나 잘못된 토큰 | 자동 갱신 시도 후 실패 시 로그인 페이지로 |
| `USER_NOT_FOUND` | 401 | 존재하지 않는 계정 | "이메일 또는 비밀번호가 올바르지 않습니다." |
