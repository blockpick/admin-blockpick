# 블록체인 관리

> 버전: 1.0 | 작성일: 2026-02-24

---

## 1. 개요

- **목적**: 블록체인 스마트 컨트랙트 배포, 트랜잭션 조회, 컨트랙트 검증 및 참여 상태 모니터링
- **대상 사용자**: ADMIN 역할 관리자
- **관련 화면**: `/blockchain` (미구현)
- **블록체인 네트워크**: Polygon Amoy (testnet) / Polygon (mainnet)

---

## 2. 화면 구성

| 화면 | 경로 | 주요 기능 | 구현 |
|------|------|----------|------|
| 블록체인 대시보드 | `/blockchain` | 컨트랙트 배포 현황, 트랜잭션 목록 | ❌ |
| 컨트랙트 상세 | `/blockchain/games/{id}` | 게임별 컨트랙트 정보 | ❌ |
| 트랜잭션 조회 | `/blockchain/transactions` | TX Hash 검색 및 상세 | ❌ |
| 엔트리 상태 조회 | `/blockchain/entries` | 참여 요청 블록체인 처리 상태 | ❌ |

> **현재 상태**: API 서비스 레이어 전량 구현 완료, UI 페이지 없음. 게임 생성은 게임 관리 화면(/games)에서 통합 처리.

## 4. 상세 기능

### 4.1 게임 생성 + 컨트랙트 배포 (`blockchainService.createGameWithBlockchain()`)

**설명**: 새 게임을 생성하면서 동시에 Polygon 네트워크에 스마트 컨트랙트를 배포. BlockPick에서 게임을 생성하는 유일한 방법.

**API:** `POST /admin/blockchain/games`

**Request Body:**
```json
{
  "gameInput": {
    "title": "게임 제목",
    "description": "게임 설명",
    "type": "DAILY",
    "category": "카테고리",
    "entryFee": 1000,
    "currency": "CASH",
    "minEntries": 1,
    "maxEntries": 100,
    "maxEntriesPerUser": 1,
    "rewardPoint": 500,
    "gridRows": 10,
    "gridCols": 10,
    "visibleFrom": "ISO8601",
    "startTime": "ISO8601",
    "endTime": "ISO8601",
    "allowDuplicate": true,
    "enableNotification": true,
    "isRecommended": false,
    "customRules": null,
    "autoEndOnMax": true,
    "autoEndOnTime": true
  },
  "deployContract": true
}
```

**Response:** `GameCreationResult` (success, message, game 포함)

**비즈니스 규칙:**
- `deployContract: true` 설정 시 컨트랙트 배포 비동기 처리
- 배포 완료까지 게임 상태 SCHEDULED 유지
- 배포 완료 후 `contract-deployed` API로 상태 전환
- 배포 실패 시 `contract-deployment-failed` API로 FAILED 처리

**UI 위치:** 게임 관리 `/games` 페이지의 "Add Game" 다이얼로그 (구현 완료)

---

### 4.2 컨트랙트 배포 완료 처리 (`blockchainService.handleContractDeployed()`)

**설명**: 외부 블록체인 리스너(또는 수동)가 컨트랙트 배포 완료를 감지하면 게임 상태를 업데이트.

**API:** `POST /admin/blockchain/games/{gameId}/contract-deployed`

**Request Body:**
```json
{
  "contractAddress": "0x1234567890abcdef...",
  "deploymentTxHash": "0xabcdef1234567890..."
}
```

**비즈니스 규칙:**
- 배포 완료 후 게임에 `onchainContractAddr`, `onchainTxHash` 저장
- 게임 상태를 SCHEDULED → IN_PROGRESS 전환 (또는 예약된 startTime까지 대기)
- 일반적으로 자동화된 블록체인 리스너가 호출하나, 관리자 수동 처리도 가능해야 함

---

### 4.3 컨트랙트 배포 실패 처리 (`blockchainService.handleContractDeploymentFailed()`)

**설명**: 컨트랙트 배포 실패 시 게임 상태를 FAILED로 변경.

**API:** `POST /admin/blockchain/games/{gameId}/contract-deployment-failed`

**Request Body:**
```json
{
  "errorCode": "DEPLOYMENT_FAILED",
  "errorMessage": "Gas estimation failed"
}
```

**비즈니스 규칙:**
- 게임 상태 SCHEDULED → FAILED
- 관리자가 실패 사유 확인 후 수동 재시도 가능

---

### 4.4 참여 블록체인 상태 조회 (`blockchainService.getEntryStatus()`)

**설명**: 특정 사용자 참여 요청의 블록체인 처리 상태 확인.

**API:** `GET /admin/blockchain/entries/{entryId}/status`

**Response:**
```json
{
  "entryId": "uuid",
  "status": "CONFIRMED",
  "txHash": "0x...",
  "errorCode": null,
  "errorMessage": null
}
```

**TxStatus 값:**
| 상태 | 설명 |
|------|------|
| `PENDING` | 블록체인 전송 대기 중 |
| `SENT` | 블록체인에 전송됨 |
| `CONFIRMED` | 컨펌 완료 |
| `FAILED` | 처리 실패 |
| `REFUNDED` | 환불 처리됨 |

---

### 4.5 컨트랙트 수동 검증 (`blockchainService.verifyContract()`)

**설명**: Polygonscan에서 배포된 컨트랙트를 수동으로 소스코드 검증.

**API:** `POST /admin/blockchain/games/{gameId}/verify-contract`

**Response:**
```json
{
  "success": true,
  "message": "Contract verified successfully",
  "verificationUrl": "https://amoy.polygonscan.com/address/0x...",
  "guid": "verification-guid"
}
```

**비즈니스 규칙:**
- 검증 완료 시 Polygonscan에서 소스코드 공개
- 실패 시 에러 메시지와 함께 수동 재시도 안내

---

### 4.6 컨트랙트 정보 조회 (`blockchainService.getContractInfo()`)

**설명**: 게임별 배포된 컨트랙트의 주소, 네트워크, 상태 정보 조회.

**API:** `GET /admin/blockchain/games/{gameId}/contract`

---

## 5. 배포 흐름

```
[관리자: 게임 생성 다이얼로그]
           |
           v
POST /admin/blockchain/games (deployContract: true)
           |
      -----+------
      |           |
   게임 생성     컨트랙트 배포 요청
      |     (비동기, Polygon 네트워크)
      v           |
 SCHEDULED  ------+------
                  |           |
              배포 성공     배포 실패
                  |           |
                  v           v
    contract-deployed  contract-deployment-failed
                  |           |
                  v           v
           SCHEDULED/IN_PROGRESS  FAILED
```

## 6. 에러 처리

| 에러 코드 | HTTP 상태 | 원인 | UI 처리 |
|----------|----------|------|--------|
| `GAME_NOT_FOUND` | 404 | 존재하지 않는 게임 | "게임을 찾을 수 없습니다." |
| `CONTRACT_ALREADY_DEPLOYED` | 400 | 이미 배포된 컨트랙트 | "컨트랙트가 이미 배포되었습니다." |
| `DEPLOYMENT_FAILED` | 500 | 배포 실패 | "컨트랙트 배포 중 오류가 발생했습니다." |
| `ENTRY_NOT_FOUND` | 404 | 존재하지 않는 엔트리 | "참여 정보를 찾을 수 없습니다." |
