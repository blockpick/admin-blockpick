# 게임 관리

> 버전: 1.0 | 작성일: 2026-02-24

---

## 1. 개요

- **목적**: BlockPick 게임의 생성, 조회, 수정, 삭제, 강제종료 및 정산 관리
- **대상 사용자**: ADMIN 역할 관리자
- **관련 화면**: `/games`, `/games/[id]/products`, `/games/[id]/leaderboard`

---

## 2. 화면 구성

| 화면 | 경로 | 주요 기능 |
|------|------|----------|
| 게임 목록 | `/games` | 검색/필터/페이지네이션, 통계 카드, CRUD 액션 |
| 게임 상세 모달 | `/games` (모달) | 게임 전체 정보 표시 |
| 게임 생성 다이얼로그 | `/games` (다이얼로그) | 멀티 스텝 (Step1: 게임 정보 → Step2: 상품 연결) |
| 게임 수정 다이얼로그 | `/games` (다이얼로그) | 게임 정보 수정 |
| 게임 강제 종료 다이얼로그 | `/games` (다이얼로그) | 강제 종료 확인 |
| 게임 상품 관리 | `/games/[id]/products` | 게임-상품 연동 관리 |
| 리더보드 | `/games/[id]/leaderboard` | 게임 참여자 순위 (미구현) |

---

## 4. 상세 기능

### 4.1 게임 목록 조회 (`gameService.getGames()`)

**설명**: 페이지네이션 지원 게임 목록. Spring Page 형식 응답.

**Query Parameters:**
| 파라미터 | 기본값 | 설명 |
|---------|--------|------|
| `page` | 0 | 페이지 번호 (0-based) |
| `size` | 20 | 페이지 크기 |
| `sortBy` | `createdAt` | 정렬 기준 |
| `sortDirection` | `DESC` | 정렬 방향 |

**GameDto 주요 필드:**
```json
{
  "id": "uuid",
  "title": "게임 제목",
  "type": "DAILY",
  "category": "카테고리",
  "status": "IN_PROGRESS",
  "entryFee": 1000,
  "currency": "CASH",
  "minEntries": 1,
  "maxEntries": 100,
  "maxEntriesPerUser": 1,
  "rewardPoint": 500,
  "gridRows": 10,
  "gridCols": 10,
  "visibleFrom": "ISO8601",
  "startTime": "ISO8601",
  "endTime": "ISO8601",
  "allowDuplicate": true,
  "enableNotification": true,
  "isRecommended": false,
  "autoEndOnMax": true,
  "autoEndOnTime": true,
  "onchainTxHash": "0x...",
  "onchainContractAddr": "0x...",
  "hasInstantPrize": false,
  "gameProducts": [...]
}
```

**UI 필터 옵션:**
| 필터 | 옵션 |
|------|------|
| 유형 | 전체 / DAILY / SELECT / VIBE / PRIME |
| 상태 | 전체 / SCHEDULED / IN_PROGRESS / PAUSED / SETTLING / ENDED / FAILED |
| 카테고리 | 전체 / 카테고리 목록 |
| 날짜 범위 | 생성일 또는 게임 시작일 기준 |
| 검색 | 게임 제목 키워드 |

**테이블 컬럼:** Thumbnail, Title, Type, Category, Status, EntryFee, Created, Actions

---

### 4.2 게임 통계 (`gameService.getGameStats()`)

**설명**: 상태별·유형별 게임 수 집계. 통계 카드에 표시.

**출력:**
```json
{
  "statusStats": {
    "SCHEDULED": 5,
    "IN_PROGRESS": 3,
    "ENDED": 50,
    "FAILED": 2
  },
  "typeStats": {
    "DAILY": 30,
    "SELECT": 15,
    "VIBE": 10,
    "PRIME": 5
  }
}
```

**통계 카드 (UI):**
| 카드 | 설명 |
|------|------|
| 총 게임 수 | 전체 게임 수 |
| 활성 게임 | IN_PROGRESS 상태 |
| 예정된 게임 | SCHEDULED 상태 |
| 종료된 게임 | ENDED 상태 |

---

### 4.3 게임 수정 (`gameService.updateGame()`)

**설명**: 게임 정보를 부분 업데이트. `id` 필드 필수.

**Request Body (UpdateGameInput):**
```json
{
  "id": "uuid (필수)",
  "title": "새 제목",
  "status": "IN_PROGRESS",
  "entryFee": 1000,
  "startTime": "ISO8601",
  "endTime": "ISO8601"
}
```

**수정 가능 필드:** title, description, mainProductName, type, category, status, entryFee, currency, minEntries, maxEntries, maxEntriesPerUser, rewardPoint, gridRows, gridCols, visibleFrom, startTime, endTime, allowDuplicate, enableNotification, isRecommended, customRules, autoEndOnMax, autoEndOnTime, onchainTxHash, onchainContractAddr

---

### 4.4 게임 삭제 (`gameService.deleteGame()`)

**설명**: UUID로 게임 삭제.

**플로우:**
1. Actions에서 "Delete" 클릭
2. 확인 다이얼로그 표시
3. 확인 시 DELETE 요청
4. 성공 시 목록에서 제거

**비즈니스 규칙:**
- ENDED/FAILED 상태 게임만 삭제 권장 (강제 삭제는 관리자 판단)

---

### 4.5 게임 강제 종료 (`gameService.forceEndGame()`)

**설명**: 진행 중인 게임을 관리자 권한으로 즉시 종료.

**비즈니스 규칙:**
- IN_PROGRESS, SCHEDULED 상태 게임에만 적용 가능
- 강제 종료 후 정산(`settle`) 별도 실행 필요
- 확인 다이얼로그 필수

---

### 4.6 게임 정산 (`gameService.settleGame()` - API 구현, UI 미구현)

**설명**: 게임 종료 후 당첨자를 선정하고 보상을 지급하는 정산 처리.

**API:** `POST /admin/games/{id}/settle`

**비즈니스 규칙:**
- ENDED 상태 게임에 대해서만 실행 가능
- 정산 실패 시 `SETTLEMENT_FAILED` 에러 반환
- UI에서 게임 상세 또는 목록에서 "정산" 버튼 제공 필요 (현재 미구현)

---

### 4.7 게임 리더보드 (`gameService.getGameLeaderboard()` - API 구현, UI 미구현)

**설명**: 게임별 참여자 순위 조회.

**API:** `GET /admin/games/{id}/leaderboard`

**표시 정보:**
- 순위, 사용자 정보 (이메일, 닉네임), 점수, 참가 횟수, 최근 참가 시간

**미구현 페이지:** `/games/[id]/leaderboard` 라우트 및 UI 미구현

## 5. 상태 다이어그램

```
[게임 생성 (블록체인)]
         |
         v
    SCHEDULED -----> IN_PROGRESS -----> SETTLING -----> ENDED
         |                 |                               |
         |         [강제종료 가능]                    [정산 실행]
         |                 |
         v                 v
      FAILED            PAUSED -----> IN_PROGRESS (재개)
```

**GameStatus 전체 값:**
| 상태 | 설명 |
|------|------|
| `SCHEDULED` | 예정됨 |
| `IN_PROGRESS` | 진행 중 |
| `PAUSED` | 일시 중지 |
| `SETTLING` | 정산 중 |
| `ENDED` | 종료 |
| `FAILED` | 실패 |

## 6. 에러 처리

| 에러 코드 | HTTP 상태 | 원인 | UI 처리 |
|----------|----------|------|--------|
| `GAME_NOT_FOUND` | 404 | 존재하지 않는 게임 ID | "게임을 찾을 수 없습니다." |
| `INVALID_GAME_STATUS` | 400 | 허용되지 않는 상태 전환 | "현재 상태에서는 이 작업을 수행할 수 없습니다." |
| `SETTLEMENT_FAILED` | 400 | 정산 처리 실패 | "정산 처리 중 오류가 발생했습니다." |
| `INTERNAL_ERROR` | 500 | 서버 내부 오류 | "서버 오류가 발생했습니다. 잠시 후 재시도하세요." |
