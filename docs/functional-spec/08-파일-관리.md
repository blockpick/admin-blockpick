# 파일 관리

> 버전: 1.0 | 작성일: 2026-02-24

---

## 1. 개요

- **목적**: 상품 이미지, 게임 썸네일, 사용자 프로필 이미지 등 관리 시스템에서 사용하는 파일을 S3에 업로드·관리
- **대상 사용자**: ADMIN 역할 관리자
- **관련 화면**: 직접 파일 관리 페이지 없음. 각 기능(상품 생성, 사용자 생성 등) 다이얼로그 내 이미지 업로드 컴포넌트로 통합 사용
- **스토리지**: AWS S3 (`blockpick-dev-storage`)

---

|------|------|----------|------|

| 상품 생성 이미지 업로드 | `/products` (다이얼로그 내) | 단일 이미지 업로드 | ✅ |
| 게임 썸네일 업로드 | `/games` (다이얼로그 내) | 단일 이미지 업로드 | ✅ |

## 4. 상세 기능

### 4.1 단일 이미지 업로드 (`storageService.uploadImage()`)

**설명**: 이미지 파일 1장을 S3에 업로드하고, 자동으로 썸네일을 생성하여 두 URL 모두 반환.

**API:** `POST /admin/storage/upload/image`
**Content-Type:** `multipart/form-data`

**Form Parameters:**
| 파라미터 | 타입 | 필수 | 설명 |
|---------|------|------|------|
| `file` | MultipartFile | O | 이미지 파일 |
| `folder` | String | X | 저장 폴더명 (기본: `products`) |

**지원 형식:** JPG, JPEG, PNG, GIF, WEBP

**Response (200):**
```json
{
  "url": "https://blockpick-dev-storage.s3.us-west-2.amazonaws.com/products/image.jpg",
  "thumbnailUrl": "https://blockpick-dev-storage.s3.us-west-2.amazonaws.com/products/image_thumb.jpg",
  "message": "업로드 성공"
}
```

**폴더 관례:**
| 폴더명 | 사용처 |
|--------|--------|
| `products` | 상품 이미지 (기본값) |
| `games` | 게임 썸네일 |
| `users` | 사용자 프로필 이미지 |

**비즈니스 규칙:**
- 업로드 성공 시 반환된 `url`을 해당 상품/게임/사용자의 이미지 필드에 저장
- 썸네일은 서버에서 자동 생성 (`_thumb` 접미사)
- 파일 크기 제한: 서버 설정에 따름 (일반적으로 10MB)

**UI 플로우 (상품 생성 예시):**
1. 파일 선택 버튼 클릭 또는 드래그앤드롭
2. `multipart/form-data`로 `POST /admin/storage/upload/image` 호출
3. 업로드 진행률 표시
4. 성공 시 이미지 미리보기 표시
5. 반환된 `url`을 `defaultImage` 또는 `thumbnail` 필드에 자동 설정
6. 상품 생성 시 해당 URL 포함하여 `POST /admin/products` 호출

### 4.2 다중 이미지 업로드 (`storageService.uploadImages()`)

**설명**: 여러 이미지 파일을 한 번에 일괄 업로드.

**API:** `POST /admin/storage/upload/images`
**Content-Type:** `multipart/form-data`

**Form Parameters:**
| 파라미터 | 타입 | 필수 | 설명 |
|---------|------|------|------|
| `files` | List\<MultipartFile\> | O | 이미지 파일들 |
| `folder` | String | X | 저장 폴더명 (기본: `products`) |

**Response (200):**
```json
{
  "results": [
    {
      "fileName": "image1.jpg",
      "url": "https://...",
      "thumbnailUrl": "https://..."
    },
    {
      "fileName": "image2.png",
      "url": "https://...",
      "thumbnailUrl": "https://..."
    }
  ],
  "errors": [],
  "successCount": 2,
  "errorCount": 0
}
```

**비즈니스 규칙:**
- 일부 파일 실패 시 나머지 파일은 정상 업로드 처리
- `errors` 배열에 실패한 파일명과 사유 포함

---

### 4.3 파일 삭제 (`storageService.deleteFile()`)

**설명**: S3에서 특정 파일을 삭제.

**API:** `DELETE /admin/storage/files/{fileId}`

**비즈니스 규칙:**
- 삭제 시 해당 파일을 참조하는 상품/게임 이미지 URL이 깨질 수 있음
- 삭제 전 참조 여부 확인 권장

---

### 4.4 파일 목록 조회 (`storageService.getFiles()`)

**설명**: 업로드된 파일 목록 조회. 폴더별 필터링 지원.

**API:** `GET /admin/storage/files`

**미구현 UI 표시 정보:**
| 항목 | 설명 |
|------|------|
| 파일명 | 원본 파일명 |
| URL | S3 접근 URL |
| 썸네일 | 썸네일 이미지 미리보기 |
| 폴더 | 저장된 폴더명 |
| 업로드 일시 | 업로드된 날짜/시각 |

---

### 4.5 파일 정보 조회 (`storageService.getFileInfo()`)

**설명**: 특정 파일의 메타데이터(크기, 타입, 업로드 일시 등) 조회.

**API:** `GET /admin/storage/files/{fileId}`

---

## 5. 에러 처리

| 에러 코드 | HTTP 상태 | 원인 | UI 처리 |
|----------|----------|------|--------|
| `INVALID_FILE_TYPE` | 400 | 지원하지 않는 파일 형식 | "JPG, PNG, GIF, WEBP 파일만 업로드 가능합니다." |
| `FILE_SIZE_EXCEEDED` | 400 | 파일 크기 초과 | "파일 크기가 너무 큽니다. (최대 10MB)" |
| `FILE_NOT_FOUND` | 404 | 존재하지 않는 파일 ID | "파일을 찾을 수 없습니다." |
| `UPLOAD_FAILED` | 500 | S3 업로드 실패 | "파일 업로드 중 오류가 발생했습니다." |
