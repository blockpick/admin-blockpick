# 시스템 모니터링

> 버전: 1.0 | 작성일: 2026-02-24

---

## 1. 개요

- **목적**: 블록체인 통합 시스템의 헬스 상태, 이벤트 발행 현황, 트랜잭션 처리 상태를 실시간 모니터링하고 실패한 이벤트를 수동으로 재발행
- **대상 사용자**: ADMIN 역할 관리자
- **관련 화면**: `/monitoring` (미구현)

---

## 2. 화면 구성

| 화면 | 경로 | 주요 기능 | 구현 |
|------|------|----------|------|
| 모니터링 대시보드 | `/monitoring` | 시스템 헬스, 이벤트/트랜잭션 상태 요약 | ❌ |
| 이벤트 목록 | `/monitoring/events` | 미발행 이벤트 목록, 재발행 기능 | ❌ |
| 트랜잭션 목록 | `/monitoring/transactions` | 트랜잭션 상태별 목록, 실패 내역 | ❌ |

> **현재 상태**: API 서비스 레이어 전량 구현 완료 (`monitoringService`). 대시보드 홈(`/`)에서 헬스 상태 카드는 표시 중.

## 4. 상세 기능

### 4.1 시스템 헬스 체크 (`monitoringService.getHealthStatus()`)

**설명**: 블록체인 통합 시스템 전체의 건강 상태를 종합 조회. 미발행 이벤트, 대기 트랜잭션, 실패 트랜잭션 수치 포함.

**API:** `GET /admin/monitoring/health`

**Response:**
```json
{
  "status": "HEALTHY",
  "timestamp": "2026-01-27T17:33:19+09:00",
  "metrics": {
    "unpublishedEvents": 0,
    "pendingTransactions": 2,
    "sentTransactions": 150,
    "failedTransactions": 1
  },
  "alerts": [
    {
      "level": "CRITICAL",
      "message": "High number of failed transactions: 11",
      "component": "TransactionProcessor"
    }
  ]
}
```

**status 값:**
| 값 | 설명 |
|----|------|
| `HEALTHY` | 정상 운영 중 |
| `DEGRADED` | 일부 성능 저하 또는 경고 존재 |

**alert level 값:**
| 값 | 설명 |
|----|------|
| `WARNING` | 주의 필요 (임박한 문제) |
| `CRITICAL` | 즉각 조치 필요 |

**비즈니스 규칙:**
- 대시보드 홈에서 주기적으로 폴링 (30초~60초 간격 권장)
- `alerts` 배열에 CRITICAL 항목 존재 시 붉은 배너 표시
- `DEGRADED` 상태 시 경고 색상으로 상태 카드 표시

**현재 UI (대시보드):**
- 시스템 상태 카드: HEALTHY/DEGRADED 표시
- metrics 수치 카드 표시 (unpublishedEvents, pendingTransactions 등)

---

### 4.2 이벤트 발행 상태 (`monitoringService.getEventStatus()`)

**설명**: Outbox 패턴으로 관리되는 이벤트의 미발행 현황 조회.

**API:** `GET /admin/monitoring/events`

**Response:**
```json
{
  "unpublishedCount": 3,
  "recentUnpublishedEvents": [
    {
      "id": "uuid",
      "aggregateType": "GAME",
      "aggregateId": "game-uuid",
      "eventType": "GAME_CREATED",
      "occurredAt": "2026-01-27T17:33:19+09:00",
      "published": false
    }
  ]
}
```

**표시 정보 (미구현 UI):**
| 항목 | 설명 |
|------|------|
| 미발행 이벤트 수 | unpublishedCount |
| 이벤트 유형 | aggregateType + eventType |
| 발생 시각 | occurredAt |
| 재발행 버튼 | 각 이벤트 행에 "Retry" 버튼 |

**비즈니스 규칙:**
- `unpublishedCount > 0` 시 경고 표시
- 장시간 미발행 이벤트는 CRITICAL 알림으로 escalation

---

### 4.3 트랜잭션 처리 상태 (`monitoringService.getTransactionStatus()`)

**설명**: 블록체인 트랜잭션의 상태별 집계 및 최근 실패 내역 조회.

**API:** `GET /admin/monitoring/transactions`

**Response:**
```json
{
  "statusCounts": {
    "PENDING": 5,
    "SENT": 150,
    "CONFIRMED": 1000,
    "FAILED": 3,
    "REFUNDED": 1
  },
  "recentFailures": [
    {
      "intentId": "intent-uuid",
      "entryId": "entry-uuid",
      "errorCode": "GAS_ESTIMATION_FAILED",
      "errorMessage": "Gas estimation failed",
      "failedAt": "2026-01-27T17:33:19+09:00"
    }
  ]
}
```

**표시 정보 (미구현 UI):**
| 항목 | 설명 |
|------|------|
| 상태별 집계 카드 | PENDING/SENT/CONFIRMED/FAILED/REFUNDED 수 |
| 실패 내역 테이블 | intentId, entryId, 에러코드, 실패 시각 |

**비즈니스 규칙:**
- `FAILED` 수가 임계값 초과 시 CRITICAL 알림
- 실패 트랜잭션별 사용자 환불 처리 여부 확인 필요

---

### 4.4 이벤트 재발행 (`monitoringService.retryEvent()`)

**설명**: 미발행 또는 실패한 이벤트를 수동으로 재발행 요청.

**API:** `POST /admin/monitoring/events/{eventId}/retry`

**Response:**
```json
{
  "message": "Event retry requested",
  "eventId": "uuid"
}
```

**플로우 (미구현 UI):**
1. 이벤트 목록에서 미발행 이벤트 선택
2. "Retry" 버튼 클릭
3. 확인 다이얼로그 표시
4. 확인 시 retry API 호출
5. 성공 시 이벤트 목록 새로고침

**비즈니스 규칙:**
- 재발행은 멱등성 보장 (중복 발행 방지)
- 재발행 후 상태가 즉시 변경되지 않을 수 있음 (비동기 처리)

---

## 5. 상태 다이어그램

```
[시스템 상태]
HEALTHY -----> DEGRADED (이상 감지 시)
   ^                |
   |                v
   +-------- [이슈 해결]

[이벤트 상태]
unpublished -----> published (정상 발행)
     |
     v
  [재발행 요청] -----> published (성공)
                  |
                  v
              [재발행 실패] (관리자 수동 조치 필요)

[트랜잭션 상태]
PENDING --> SENT --> CONFIRMED
  |
  v
FAILED --> REFUNDED
```

## 6. 에러 처리

| 에러 코드 | HTTP 상태 | 원인 | UI 처리 |
|----------|----------|------|--------|
| `EVENT_NOT_FOUND` | 404 | 존재하지 않는 이벤트 ID | "이벤트를 찾을 수 없습니다." |
| `RETRY_FAILED` | 500 | 재발행 처리 실패 | "이벤트 재발행 중 오류가 발생했습니다." |
| `EVENT_ALREADY_PUBLISHED` | 400 | 이미 발행된 이벤트 재발행 시도 | "이미 발행된 이벤트입니다." |
