# 대시보드

> 버전: 1.0 | 작성일: 2026-02-24

---

## 1. 개요

- **목적**: 관리자가 BlockPick 시스템 전체 현황을 한눈에 파악할 수 있는 종합 통계 화면
- **대상 사용자**: ADMIN, PARTNER 역할 관리자
- **관련 화면**: `/` (루트, 대시보드 홈)

---

## 2. 화면 구성

| 화면 | 경로 | 주요 기능 |
|------|------|----------|
| 대시보드 홈 | `/` | 종합 통계, 차트, 활동 내역, 시스템 헬스 |

**레이아웃 구조:**
```
대시보드 페이지
├── 통계 카드 행 (4개)
│   ├── 총 사용자 수
│   ├── 총 게임 수
│   ├── 총 상품 수
│   └── 시스템 상태 (HEALTHY/DEGRADED)
├── 차트 섹션 (2x2 그리드)
│   ├── 사용자 가입 추이
│   ├── 게임 생성 추이
│   ├── 수익 추이
│   └── 상품 등록 추이
├── 활동 내역 타임라인
└── 최근 게임 / 최근 사용자 목록
```

---

## 3. API 목록

| # | API명 | 메서드 | 경로 | 설명 | 구현 |
|---|-------|--------|------|------|------|
| 1 | 대시보드 통계 | GET | `/admin/dashboard/stats` | 종합 통계 수치 | ✅ |
| 2 | 차트 데이터 | GET | `/admin/dashboard/charts/{type}` | 유형별 시계열 차트 데이터 | ✅ |
| 3 | 최근 활동 | GET | `/admin/dashboard/activities` | 최근 활동 내역 목록 | ✅ |

## 4. 상세 기능

### 4.1 대시보드 통계 (`dashboardService.getStats()`)

**설명**: 사용자, 게임, 상품, 시스템 관련 핵심 지표를 한 번에 조회.

**API:** `GET /admin/dashboard/stats`

**통계 카드 (UI):**
| 카드 | 설명 | 연결 페이지 |
|------|------|------------|
| 총 사용자 수 | 전체 사용자 수 및 증감률 | `/users` |
| 총 게임 수 | 전체 게임 수 및 현재 진행 중 | `/games` |
| 총 상품 수 | 전체 상품 수 및 활성 상품 수 | `/products` |
| 시스템 상태 | HEALTHY/DEGRADED + 알림 수 | 모니터링 섹션 |

**비즈니스 규칙:**
- 각 카드는 숫자 외 증감률(%) 또는 하위 카테고리 수치 함께 표시
- 카드 클릭 시 해당 관리 페이지로 이동

---

### 4.2 차트 데이터 (`dashboardService.getChartData()`)

**설명**: 기간별 시계열 데이터를 차트 형태로 표시.

**API:** `GET /admin/dashboard/charts/{type}`

**지원 차트 유형 (`{type}`):**
| 유형 | 설명 |
|------|------|
| `users` | 사용자 가입 추이 |
| `games` | 게임 생성 추이 |
| `revenue` | 수익 추이 |
| `products` | 상품 등록 추이 |

**UI 기간 선택:**
| 옵션 | 설명 |
|------|------|
| `day` | 최근 24시간 (시간별) |
| `week` | 최근 7일 (일별) |
| `month` | 최근 30일 (일별) |
| `year` | 최근 12개월 (월별) |

**차트 컴포넌트:** `ChartCard` (라인 차트, 기간 선택 탭 포함)

**비즈니스 규칙:**
- 기간 변경 시 해당 차트만 데이터 재요청
- 4개 차트 2x2 그리드 배치
- X축: 날짜/시간, Y축: 수치

---

### 4.3 최근 활동 내역 (`dashboardService.getRecentActivities()`)

**설명**: 사용자/게임/상품 관련 최근 활동 타임라인 표시.

**API:** `GET /admin/dashboard/activities`

**활동 표시 항목:**
| 활동 유형 | 아이콘/색상 | 예시 |
|----------|------------|------|
| 사용자 생성 | 파란색 사람 아이콘 | "새 사용자 kim@example.com 가입" |
| 게임 생성 | 초록색 게임 아이콘 | "새 게임 '데일리챌린지 #42' 생성" |
| 게임 종료 | 회색 체크 아이콘 | "게임 '위클리럭키' 종료" |
| 상품 등록 | 주황색 상자 아이콘 | "새 상품 'AirPods Pro' 등록" |
| 시스템 알림 | 빨간색 경고 아이콘 | "트랜잭션 처리 실패 감지" |

**비즈니스 규칙:**
- 최근 10개 활동 표시 (무한 스크롤 미구현)
- 각 활동 항목은 관련 리소스 페이지로 클릭 가능한 링크 제공
- 타임라인 형식: 좌측 타임스탬프, 우측 활동 내용

---

### 4.4 최근 게임/사용자 목록

**설명**: 대시보드 하단에 최근 생성된 게임과 사용자 목록을 미리보기 형태로 표시.

**최근 게임 목록:**
| 컬럼 | 설명 |
|------|------|
| 제목 | 게임 제목 |
| 유형 | DAILY/SELECT/VIBE/PRIME |
| 상태 | SCHEDULED/IN_PROGRESS/ENDED 등 |
| 생성일 | ISO 날짜 |

**최근 사용자 목록:**
| 컬럼 | 설명 |
|------|------|
| 아바타 | 프로필 이미지 또는 이니셜 |
| 이메일 | 이메일 주소 |
| 역할 | USER/ADMIN/PARTNER |
| 가입일 | ISO 날짜 |

**비즈니스 규칙:**
- 최근 5~10개 항목 표시
- "전체 보기" 버튼으로 `/games` 또는 `/users` 페이지 이동

---

### 4.5 시스템 헬스 상태 표시

**설명**: 모니터링 API 데이터를 대시보드 통계 카드에 통합 표시.

**데이터 출처:** `GET /admin/monitoring/health`

**표시 항목:**
| 항목 | 설명 |
|------|------|
| 시스템 상태 | HEALTHY (녹색) / DEGRADED (주황) |
| 미발행 이벤트 수 | `metrics.unpublishedEvents` |
| 대기 트랜잭션 수 | `metrics.pendingTransactions` |
| 실패 트랜잭션 수 | `metrics.failedTransactions` |
| 알림 수 | `alerts` 배열 크기 |

---

## 5. 에러 처리

| 에러 코드 | HTTP 상태 | 원인 | UI 처리 |
|----------|----------|------|--------|
| `UNAUTHORIZED` | 401 | 인증 토큰 없음/만료 | 로그인 페이지 리다이렉트 |
| `STATS_UNAVAILABLE` | 503 | 통계 서비스 일시 불가 | "통계 데이터를 불러올 수 없습니다." + 재시도 버튼 |
| `CHART_DATA_ERROR` | 500 | 차트 데이터 조회 실패 | 해당 차트 에러 상태 표시, 나머지 차트 정상 표시 |
