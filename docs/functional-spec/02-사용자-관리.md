# 사용자 관리

> 버전: 1.0 | 작성일: 2026-02-24

---

## 1. 개요

- **목적**: BlockPick 앱 사용자의 전체 목록 조회, 생성, 수정, 삭제 및 역할 관리
- **대상 사용자**: ADMIN 역할 관리자
- **관련 화면**: `/users`

---

## 2. 화면 구성

| 화면 | 경로 | 주요 기능 |
|------|------|----------|
| 사용자 목록 | `/users` | 검색/필터/페이지네이션, 통계 카드, CRUD 액션 |
| 사용자 상세 모달 | `/users` (모달) | 사용자 상세 정보 표시 |
| 사용자 생성 다이얼로그 | `/users` (다이얼로그) | 신규 사용자 생성 |
| 사용자 수정 다이얼로그 | `/users` (다이얼로그) | 기존 사용자 정보 수정 |
| 역할 변경 다이얼로그 | `/users` (다이얼로그) | 사용자 역할 변경 |

---

## 3. API 목록

| # | API명 | 메서드 | 경로 | 설명 | 구현 |
|---|-------|--------|------|------|------|
| 1 | 사용자 목록 조회 | GET | `/admin/users` | 전체 사용자 목록 (현재 페이지네이션 미지원) | ✅ |
| 2 | 사용자 상세 조회 | GET | `/admin/users/{id}` | 특정 사용자 상세 정보 | ✅ |
| 3 | 사용자 생성 | POST | `/admin/users` | 신규 사용자 직접 생성 (역할 지정 가능) | ✅ |
| 4 | 사용자 수정 | PUT | `/admin/users/{id}` | 기존 사용자 정보 수정 (partial update) | ✅ |

| 7 | 사용자 통계 | GET | `/admin/users/stats` | 상태별/역할별 통계 | ✅ |

## 4. 상세 기능

### 4.1 사용자 목록 조회 (`userService.getUsers()`)

**설명**: 전체 사용자 목록을 조회하며 검색·필터 파라미터를 지원.

**출력 (Response 200):**
```json
{
  "success": true,
  "code": "SUCCESS",
  "data": [
    {
      "id": "uuid",
      "email": "user@example.com",
      "nickname": "유저닉네임",
      "profileImageUrl": "https://...",
      "shoppingCash": 10000.0,
      "eventPoint": 5000.0,
      "participationPoint": 3000.0,
      "isPushNotification": true,
      "isMarketingNotification": false,
      "isBan": false,
      "userRole": "USER",
      "isSocialAccount": false,
      "socialProvider": null,
      "socialName": null
    }
  ],
  "count": 1
}
```

> **주의**: 현재 서버 측 페이지네이션 미지원. 전체 목록을 한 번에 반환. 대규모 데이터 시 프론트엔드 가상 스크롤 또는 클라이언트 페이지네이션 적용 필요.

**UI 필터 옵션:**
| 필터 | 옵션 |
|------|------|
| 상태 | 전체 / ACTIVE / INACTIVE / SUSPENDED / DELETED |
| 역할 | 전체 / USER / ADMIN / PARTNER |
| 가입일 | 날짜 범위 선택 |
| 검색 | 이메일/닉네임 키워드 검색 |

**비즈니스 규칙:**
- 필터/검색 변경 시 즉시 목록 업데이트
- 테이블 컬럼: Avatar, Email, Nickname, Role, Status, ShoppingCash, Created, Actions

---

### 4.2 사용자 상세 조회 (`userService.getUserById()`)

**설명**: UUID로 특정 사용자의 상세 정보를 조회하여 모달에 표시.

**표시 정보:**
- 기본 정보: 이메일, 닉네임, 역할, 가입일
- 재화 정보: shoppingCash, eventPoint, participationPoint
- 소셜 계정: isSocialAccount, socialProvider, socialName
- 계정 상태: isBan, isPushNotification, isMarketingNotification

**수용 기준:**
- 모달에서 수정/역할변경/삭제 버튼 제공
- 재화 금액은 원화(₩) 포맷으로 표시

---

### 4.3 사용자 생성 (`userService.createUser()`)

**설명**: 관리자가 직접 신규 사용자를 생성. 역할 지정 가능.

**입력 (Request Body):**
```json
{
  "email": "user@example.com",
  "password": "password123",
  "nickname": "홍길동",
  "profileImageUrl": "https://...",
  "role": "USER"
}
```

| 필드 | 타입 | 필수 | 규칙 |
|------|------|------|------|
| `email` | String | O | @ 포함 이메일 형식 |
| `password` | String | O | 최소 6자 |
| `nickname` | String | X | - |
| `profileImageUrl` | String | X | Storage API로 업로드 후 URL 사용 |
| `role` | UserRole | X | USER / ADMIN / PARTNER (기본: USER) |

**비즈니스 규칙:**
- 이메일 중복 시 에러 반환
- 생성 성공 시 목록 새로고침 및 성공 토스트 표시

---

### 4.4 사용자 수정 (`userService.updateUser()`)

**설명**: 특정 사용자의 이메일, 닉네임, 프로필 이미지를 수정 (partial update).

**입력 (Request Body):**
```json
{
  "email": "new@example.com",
  "nickname": "새닉네임",
  "profileImageUrl": "https://..."
}
```

**비즈니스 규칙:**
- 모든 필드 선택적 (변경하지 않는 필드는 생략)
- 수정 성공 시 목록 새로고침

---

### 4.5 사용자 삭제 (`userService.deleteUser()`)

**설명**: UUID로 특정 사용자를 삭제.

**플로우:**
1. Actions 메뉴에서 "Delete" 클릭
2. 확인 다이얼로그 표시 ("이 사용자를 삭제하시겠습니까?")
3. 확인 시 DELETE 요청
4. 성공 시 목록에서 제거 및 성공 토스트

**비즈니스 규칙:**
- 삭제 전 반드시 확인 다이얼로그 거쳐야 함
- 삭제된 사용자는 복구 불가

---

### 4.6 사용자 역할 변경 (`userService.updateUserRole()`)

**설명**: 사용자 역할을 변경 (이메일 기반).

**입력 (Request Body):**
```json
{
  "email": "user@example.com",
  "role": "ADMIN"
}
```

**출력 (Response 200):**
```json
{
  "success": true,
  "code": "SUCCESS",
  "message": "역할 변경 성공",
  "user": {
    "id": "uuid",
    "email": "user@example.com",
    "nickname": "닉네임",
    "role": "ADMIN",
    "updatedAt": "2026-01-27T17:33:19+09:00"
  }
}
```

**비즈니스 규칙:**
- 변경 가능 역할: USER / ADMIN / PARTNER
- 역할 변경은 이메일 기반 (ID 아님)
- 변경 즉시 목록에 반영

---

### 4.7 사용자 통계 (`userService.getUserStats()`)

**설명**: 대시보드 통계 카드에 표시할 사용자 상태별 통계.

**통계 카드 (UI):**
| 카드 | 설명 |
|------|------|
| 총 사용자 | 전체 사용자 수 |
| 활성 사용자 | ACTIVE 상태 사용자 수 |
| 오늘 가입자 | 금일 신규 가입 수 |
| 정지된 사용자 | isBan = true 또는 SUSPENDED 상태 |

---

## 5. 상태 다이어그램

```
ACTIVE <---> SUSPENDED (밴 처리/해제)
  |
  v
DELETED (삭제, 복구 불가)
```

---

## 6. 에러 처리

| 에러 코드 | HTTP 상태 | 원인 | UI 처리 |
|----------|----------|------|--------|
| `USER_NOT_FOUND` | 404 | 존재하지 않는 사용자 ID | "사용자를 찾을 수 없습니다." 토스트 |
| `EMAIL_ALREADY_EXISTS` | 409 | 이메일 중복 | 생성 다이얼로그 내 인라인 에러 |
| `INVALID_ROLE` | 400 | 잘못된 역할 값 | "올바른 역할을 선택해주세요." |
| `UNAUTHORIZED` | 401 | 인증 토큰 없음/만료 | 로그인 페이지 리다이렉트 |
