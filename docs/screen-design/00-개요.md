## 공통 레이아웃 (AdminLayout)

### 구조

```
┌─────────────────────────────────────────────┐
│ Header (h-16, border-b)                     │
├──────────┬──────────────────────────────────┤
│          │                                  │
│ Sidebar  │  Main Content                    │
│ (w-64 /  │  (flex-1, overflow-y-auto)       │
│  w-16    │  p-8                             │
│ collapse)│                                  │
│          │                                  │
└──────────┴──────────────────────────────────┘
```

### 반응형 처리

| 브레이크포인트 | 사이드바 | 헤더 |
|---------------|----------|------|
| 모바일 (< md) | 숨김 → Sheet 슬라이드 | 햄버거 버튼 노출 |
| 데스크톱 (≥ md) | 고정 노출 | 햄버거 버튼 숨김 |

### 인증 게이트웨이

- `AdminLayout` 진입 시 `useAuth()` 훅으로 인증 상태 확인
- 토큰 없음 또는 인증 실패 → `/login` 리다이렉트
- 인증 확인 중 → 전체 화면 `LoadingSpinner` 표시

## 사이드바 메뉴 구조

| 순서 | 메뉴명 | 경로 | 아이콘 |
|------|--------|------|--------|
| 1 | 대시보드 | /dashboard | LayoutDashboard |
| 2 | 사용자 관리 | /users | Users |
| 3 | 게임 관리 | /games | Gamepad2 |
| 4 | 상품 관리 | /products | Package |
| 5 | 블록체인 | /blockchain | Link |
| 6 | 파일 관리 | /storage | Folder |
| 7 | 모니터링 | /monitoring | BarChart3 |
| 8 | 설정 | /settings | Settings |
| - | 로그아웃 | - (logout API 호출) | LogOut |

### 사이드바 동작

- **펼침 상태**: `w-64`, 아이콘 + 텍스트 레이블 표시
- **접힘 상태**: `w-16`, 아이콘만 표시, 호버 시 툴팁 노출
- collapse 상태는 `localStorage('sidebar-collapsed')`에 저장하여 세션 간 유지
- Sheet 내부(모바일)에서는 항상 펼침 상태

### 활성 메뉴 판별

- `pathname === item.href` 또는 `pathname.startsWith(item.href + '/')`
- 활성 메뉴: `bg-primary text-primary-foreground`
- 비활성 메뉴: `text-muted-foreground hover:bg-accent hover:text-accent-foreground`

## 공통 패턴

### 페이지 구조 패턴

```
AdminLayout
└── div.space-y-6
    ├── PageHeader (title, description, action 버튼)
    ├── StatsCard 그리드 (보통 4열)
    ├── 필터바 (옵션)
    └── DataTable / 콘텐츠 영역
```

### 다이얼로그 패턴

모든 CUD(Create/Update/Delete) 작업은 Dialog/AlertDialog를 통해 처리하며, 성공/실패 시 `useToast()`로 알림을 표시합니다.

### 로딩 상태 패턴

- 전체 화면 로딩: `LoadingSpinner size="lg"` (flex h-screen items-center justify-center)
- 데이터 없음: `EmptyState` 컴포넌트
- 인라인 로딩: `Skeleton` 컴포넌트 (각 컴포넌트 자체 처리)
