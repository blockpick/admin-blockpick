## 탭 1: 로그 (logs)

### 필터 영역

```
div.flex.gap-4.mb-4
├── Select (로그 레벨)
│   └── 옵션: 전체 / ERROR / WARN / INFO / DEBUG
└── Select (서비스)
    └── 옵션: 전체 / 서비스 목록
```

### DataTable (로그 목록)

**컬럼 정의**:

| 컬럼 | 내용 | 정렬 |
|------|------|------|
| 레벨 | Badge (level) | 가능 |
| 서비스 | service | 가능 |
| 메시지 | message (truncate) | - |
| 시간 | timestamp | 가능 |
| 상세 | 확장 버튼 | - |

**로그 레벨 Badge**:

| 레벨 | Variant | 색상 |
|------|---------|------|
| ERROR | `destructive` | 빨간색 |
| WARN | `secondary` | 노란색 계열 |
| INFO | `default` | 기본 |
| DEBUG | `outline` | 회색 |

---

## 탭 2: 메트릭 (metrics)

`grid gap-4 md:grid-cols-1` (차트 3개 세로 배치)

| 순서 | ChartCard title | 데이터 |
|------|----------------|--------|
| 1 | CPU 사용률 | `metricsData.cpu` (%) |
| 2 | 메모리 사용률 | `metricsData.memory` (%) |
| 3 | 요청 수 | `metricsData.requests` (req/min) |

- 모두 ChartCard 컴포넌트 사용
- 기간 선택: 일/주/월/년

## 탭 3: 이벤트 (events)

## 데이터 연동

| Hook | 역할 |
|------|------|
| `useMonitoringStats()` | 통계 카드 데이터 |
| `useLogs(level, service, pagination)` | 로그 목록 |
| `useSystemMetrics(period)` | CPU/메모리/요청 메트릭 |
| `useUnpublishedEvents()` | 미발행 이벤트 목록 |
| `useFailedTransactions()` | 실패 트랜잭션 목록 |
| `usePublishEvent()` | 이벤트 수동 발행 mutation |
| `useRetryTransaction()` | 트랜잭션 재시도 mutation |

---

## 자동 갱신

- 로그 탭: 30초마다 자동 갱신 (`refetchInterval: 30000`)
- 메트릭 탭: 10초마다 자동 갱신 (`refetchInterval: 10000`)
- 이벤트/트랜잭션 탭: 수동 갱신 (새로고침 버튼)
