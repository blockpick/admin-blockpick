## 5.2 주요 DTO 구조

### UserDto

```typescript
interface UserDto {
  id: string;                        // UUID
  email: string;
  nickname: string | null;
  profileImageUrl: string | null;
  shoppingCash: number;              // 쇼핑 캐시 잔액
  eventPoint: number;                // 이벤트 포인트 잔액
  participationPoint: number;        // 참여 포인트 잔액
  isPushNotification: boolean;
  isMarketingNotification: boolean;
  isBan: boolean;                    // 밴 여부
  userRole: UserRole;                // "USER" | "ADMIN" | "PARTNER"
  isSocialAccount: boolean;
  socialProvider: SocialProvider | null;  // "KAKAO" | "NAVER" | "GOOGLE" | "FACEBOOK"
  socialName: string | null;
}
```

### GameDto

```typescript
interface GameDto {
  id: string;                        // UUID

  gridCols: number;

  endTime: string;                   // ISO 8601
  allowDuplicate: boolean;

  isRecommended: boolean;
  customRules: string | null;
```

## 5.3 Enum 정의

### GameType (게임 유형)

| 값 | 설명 |
|----|------|
| `DAILY` | 일일 게임 |
| `SELECT` | 선택형 게임 |
| `VIBE` | 바이브 게임 |
| `PRIME` | 프라임 게임 |

### GameStatus (게임 상태)

| 값 | 설명 |
|----|------|
| `SCHEDULED` | 예정됨 (컨트랙트 배포 완료 후) |
| `IN_PROGRESS` | 진행 중 |
| `PAUSED` | 일시 중지 |
| `SETTLING` | 정산 중 |
| `ENDED` | 종료 |
| `FAILED` | 실패 (컨트랙트 배포 실패 등) |

### UserRole (사용자 역할)

| 값 | 설명 |
|----|------|
| `USER` | 일반 사용자 |
| `ADMIN` | 관리자 |
| `PARTNER` | 파트너 |

### CurrencyType — 게임 입장 화폐

| 값 | 설명 |
|----|------|
| `CASH` | 현금 (쇼핑캐시) |
| `POINT` | 포인트 |

### CurrencyType — 지갑 재화

| 값 | 설명 |
|----|------|
| `SHOPPING_CASH` | 쇼핑 캐시 |
| `EVENT_POINT` | 이벤트 포인트 |
| `PARTICIPATION_POINT` | 참여 포인트 |

### TxStatus (블록체인 트랜잭션 상태)

| 값 | 설명 |
|----|------|
| `PENDING` | 전송 대기 |
| `SENT` | 블록체인에 전송됨 |
| `CONFIRMED` | 컨펌 완료 |
| `FAILED` | 실패 |
| `REFUNDED` | 환불 처리됨 |

### SocialProvider (소셜 로그인 제공자)

| 값 | 설명 |
|----|------|
| `KAKAO` | 카카오 |
| `NAVER` | 네이버 |
| `GOOGLE` | 구글 |
| `FACEBOOK` | 페이스북 |

## 5.4 상태 다이어그램

### 게임 상태 흐름

```
[생성 요청]
     │
     ▼
POST /admin/blockchain/games
     │
     ├─── 배포 성공 ──► SCHEDULED ──► IN_PROGRESS ──► SETTLING ──► ENDED
     │                      │               │
     │                      │           강제 종료
     │                      │           (force-end)
     │                      │               │
     │                 (시작 전 취소)         └──► ENDED
     │
     └─── 배포 실패 ──► FAILED
```

| 전이 | 트리거 | API |
|------|--------|-----|
| 생성 → SCHEDULED | 컨트랙트 배포 완료 | `POST .../contract-deployed` |
| 생성 → FAILED | 컨트랙트 배포 실패 | `POST .../contract-deployment-failed` |
| SCHEDULED → IN_PROGRESS | 시작 시간 도달 (자동) | - |
| IN_PROGRESS → SETTLING | 종료 시간 도달 또는 강제 종료 | `PUT .../force-end` |
| IN_PROGRESS → ENDED | 강제 종료 | `PUT .../force-end` |
| SETTLING → ENDED | 정산 완료 | `POST .../settle` |
| 모든 상태 | 수동 수정 | `PUT /admin/games` |

### 블록체인 트랜잭션 상태 흐름

```
PENDING ──► SENT ──► CONFIRMED
               │
               └──► FAILED ──► REFUNDED
```
