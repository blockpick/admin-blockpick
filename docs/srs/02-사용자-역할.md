# 2. 사용자 역할 및 권한

## 2.1 역할 정의

| 역할 | 코드 | 설명 | 생성 방법 |
|------|------|------|-----------|
| **일반 사용자** | `USER` | BlockPick 앱 이용 사용자 (기본값) | 회원가입, 관리자 직접 생성 |
| **관리자** | `ADMIN` | 플랫폼 전체 운영 권한 | 관리자가 역할 변경으로 부여 |
| **파트너** | `PARTNER` | 브랜드/벤더 파트너 | 관리자가 역할 변경으로 부여 |

> Admin Dashboard 로그인은 `ADMIN` 또는 `PARTNER` 역할만 가능하다. `USER` 역할은 로그인 시 `FORBIDDEN (403)` 에러가 반환된다.

## 2.3 인증 방식

### JWT Bearer Token

Admin Dashboard는 JWT(JSON Web Token) 기반 인증을 사용한다.

**인증 헤더 형식:**
```
Authorization: Bearer <accessToken>
```

**토큰 알고리즘:** HS256

### 로그인 플로우

```
1. 사용자가 이메일/비밀번호 입력
2. POST /admin/auth/login 호출
3. 서버에서 역할 검증 (ADMIN 또는 PARTNER만 허용)
4. 성공 시 accessToken + refreshToken 반환
5. 클라이언트에서 토큰 저장 (httpOnly Cookie 또는 localStorage)
6. 이후 모든 API 요청 시 Authorization 헤더 첨부
```

**로그인 에러 코드:**

| HTTP 상태 | 코드 | 설명 |
|-----------|------|------|
| 401 | `INVALID_CREDENTIALS` | 이메일 또는 비밀번호 불일치 |
| 403 | `FORBIDDEN` | USER 역할로 로그인 시도 |

## 2.4 세션 관리

### 토큰 만료 정책

| 토큰 종류 | 만료 시간 | 설명 |
|-----------|-----------|------|
| **Access Token** | 24시간 (86,400,000ms) | API 요청에 사용 |
| **Refresh Token** | 7일 (604,800,000ms) | Access Token 갱신에 사용 |

### 토큰 갱신 플로우

```
1. API 요청 실패 (401 Unauthorized)
2. 클라이언트에서 Refresh Token으로 갱신 시도
   POST /admin/auth/refresh
   Body: { "refreshToken": "..." }
3. 성공 시 새 Access Token 발급
4. 실패 시 (Refresh Token 만료) → 로그아웃 처리 후 로그인 페이지로 이동
```

### 토큰 검증 (페이지 로드 시)

페이지 로드 또는 새로고침 시 `POST /admin/auth/verify`를 호출하여 토큰 유효성을 검증하고 관리자 정보를 갱신한다.

**검증 에러 코드:**

| HTTP 상태 | 코드 | 설명 |
|-----------|------|------|
| 401 | `INVALID_TOKEN` | 토큰이 유효하지 않음 |
| 401 | `USER_NOT_FOUND` | 토큰에 해당하는 사용자 없음 |
| 403 | `FORBIDDEN` | 권한 없음 |

### 로그아웃 처리

- 클라이언트 측 토큰 삭제
- `POST /admin/auth/logout` 호출 (서버 측 세션 무효화)
- 로그인 페이지로 리다이렉트
