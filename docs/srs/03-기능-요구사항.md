# 3. 기능 요구사항

## FR-001: 관리자 인증

| 항목 | 내용 |
|------|------|
| **ID** | FR-001 |
| **우선순위** | High |
| **상태** | 구현완료 |
| **관련 API** | `POST /admin/auth/login`, `POST /admin/auth/logout`, `POST /admin/auth/refresh`, `POST /admin/auth/verify`, `GET /admin/auth/me` |

**기능 목록:**
- 이메일/비밀번호 로그인 (ADMIN, PARTNER 역할만 허용)
- 서버 로그아웃 및 클라이언트 토큰 삭제
- Refresh Token으로 Access Token 자동 갱신
- 페이지 로드 시 토큰 유효성 검증

**비즈니스 규칙:**
- `USER` 역할은 로그인 불가 (403 반환)
- Access Token 만료(24h) 시 Refresh Token(7일)으로 자동 갱신
- Refresh Token 만료 시 강제 로그아웃

**수용 기준 (Acceptance Criteria):**
- [ ] 올바른 관리자 계정으로 로그인 시 Dashboard로 이동한다
- [ ] 잘못된 자격증명 시 에러 메시지를 표시한다
- [ ] USER 역할 계정으로 로그인 시 권한 없음 메시지를 표시한다
- [ ] 로그아웃 시 토큰이 삭제되고 로그인 페이지로 이동한다
- [ ] 만료된 Access Token은 자동으로 갱신된다

## FR-002: 유저 관리

| 항목 | 내용 |
|------|------|
| **ID** | FR-002 |
| **우선순위** | High |
| **상태** | 구현완료 |
| **관련 API** | `GET /admin/users`, `GET /admin/users/{id}`, `POST /admin/users`, `PUT /admin/users/{id}`, `DELETE /admin/users/{id}`, `PATCH /admin/users/role`, `GET /admin/users/stats` |

**기능 목록:**
- 전체 유저 목록 조회 (테이블 표시)
- 유저 상세 정보 조회 (모달)
- 유저 생성 (이메일, 비밀번호, 닉네임, 역할)
- 유저 정보 수정 (이메일, 닉네임, 프로필 이미지)
- 유저 삭제 (확인 다이얼로그)
- 유저 역할 변경 (USER / ADMIN / PARTNER)
- 유저 통계 조회 (상태별 집계)
- 검색 및 필터링 (상태, 역할, 날짜 범위)
- 페이지네이션

**비즈니스 규칙:**
- 이메일은 고유해야 한다
- 비밀번호는 최소 6자 이상
- 역할 변경은 이메일 기반으로 처리 (`PATCH /admin/users/role` - body에 email 포함)
- 현재 `GET /admin/users`는 페이지네이션 미지원, 전체 목록 반환

**수용 기준:**
- [ ] 유저 목록이 테이블로 표시된다
- [ ] 이메일/닉네임으로 검색이 가능하다
- [ ] 상태/역할/날짜 범위로 필터링이 가능하다
- [ ] 유저 생성 다이얼로그에서 모든 필드 입력 후 저장 시 목록에 반영된다
- [ ] 유저 삭제 시 확인 다이얼로그가 표시된다
- [ ] 역할 변경 후 즉시 테이블에 반영된다
- [ ] 통계 카드(총 유저, 활성, 오늘 가입, 역할 분포)가 표시된다

## FR-003: 게임 관리

| 항목 | 내용 |

| **우선순위** | High |
| **상태** | 구현완료 (리더보드 UI 미구현) |
| **관련 API** | `GET /admin/games`, `GET /admin/games/{id}`, `PUT /admin/games`, `DELETE /admin/games/{id}`, `PUT /admin/games/{id}/force-end`, `POST /admin/games/{id}/settle`, `GET /admin/games/type/{type}`, `GET /admin/games/status/{status}`, `GET /admin/games/stats`, `GET /admin/games/search`, `GET /admin/games/recommended`, `GET /admin/games/date-range`, `GET /admin/games/{id}/leaderboard`, `POST /admin/blockchain/games` |

**기능 목록:**
- 게임 목록 조회 (페이지네이션, 정렬)
- 게임 상세 조회 (기본 정보, 블록체인 정보, 상품 목록)
- 게임 생성 (블록체인 컨트랙트 배포 포함, 멀티 스텝)
- 게임 정보 수정
- 게임 삭제 (확인 다이얼로그)
- 게임 강제 종료 (IN_PROGRESS 상태일 때)
- 게임 정산 (당첨자 선정)
- 타입별 조회 (DAILY / SELECT / VIBE / PRIME)
- 상태별 조회 (SCHEDULED / IN_PROGRESS / PAUSED / SETTLING / ENDED / FAILED)
- 제목 검색
- 날짜 범위 조회
- 추천 게임 조회
- 게임 통계 (상태별/타입별 집계)
- 리더보드 조회 (API 구현됨, UI 미구현)

**비즈니스 규칙:**
- 게임 생성은 `POST /admin/blockchain/games`를 통해 블록체인 컨트랙트 배포와 함께 처리
- `POST /admin/games` 직접 생성 엔드포인트는 존재하지 않음
- 강제 종료는 ACTIVE/IN_PROGRESS 상태 게임에만 가능
- 정산(`/settle`)은 게임 종료 후 수행

**수용 기준:**
- [ ] 게임 목록이 페이지네이션으로 표시된다
- [ ] 타입/상태/카테고리/날짜 범위 필터링이 동작한다
- [ ] 게임 생성 시 Step 1(기본 정보) → Step 2(상품 연결) 멀티 스텝이 동작한다
- [ ] 게임 상세 모달에서 블록체인 컨트랙트 정보가 표시된다
- [ ] 강제 종료 시 확인 다이얼로그가 표시되고 상태가 변경된다
- [ ] 통계 카드(총 게임, 활성, 예정, 종료)가 표시된다

| 항목 | 내용 |

| **ID** | FR-006 |

| **상태** | API 구현완료, UI 미구현 |
| **관련 API** | `POST /admin/blockchain/games`, `POST /admin/blockchain/games/{gameId}/contract-deployed`, `POST /admin/blockchain/games/{gameId}/contract-deployment-failed`, `POST /admin/blockchain/games/{gameId}/verify-contract`, `GET /admin/blockchain/entries/{entryId}/status`, `GET /admin/blockchain/transactions/{txHash}`, `GET /admin/blockchain/games/{gameId}/contract` |

**기능 목록:**
- 게임 생성 + 스마트 컨트랙트 배포 요청 (게임 생성 플로우에 통합)
- 컨트랙트 배포 완료 처리 (상태 → SCHEDULED)
- 컨트랙트 배포 실패 처리 (상태 → FAILED)
- 컨트랙트 수동 검증 (Polygonscan)
- 참여 엔트리 블록체인 상태 조회
- 트랜잭션 해시로 트랜잭션 정보 조회
- 게임별 컨트랙트 정보 조회

**비즈니스 규칙:**
- 게임 생성은 반드시 `POST /admin/blockchain/games`를 통해 처리
- `deployContract: true`로 요청 시 블록체인 컨트랙트 자동 배포 시도
- 배포 실패 시 게임 상태가 `FAILED`로 변경됨
- 컨트랙트 검증은 Polygon Amoy Testnet 기준

**수용 기준:**
- [ ] 게임 생성 시 블록체인 컨트랙트 배포가 자동으로 요청된다
- [ ] 배포 성공/실패 상태가 게임 상세에서 확인 가능하다
- [ ] 트랜잭션 해시로 트랜잭션 정보를 조회할 수 있다 (UI 미구현)
- [ ] 엔트리 상태를 조회할 수 있다 (UI 미구현)
